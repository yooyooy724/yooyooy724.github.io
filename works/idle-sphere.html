_02_／Selected Work

Idle Sphere

球体が織りなす星系のような幻想的なグラフィックの中で、 インフレーションを楽しむ放置系ゲーム。本作からレベニューシェアに切り替わり、 チームリーダーとしての立ち回りと、設計思想の限界の両方を経験したプロジェクトです。

リリース
2024年11月17日（Steam 早期アクセス）
プラットフォーム
Steam (PC・Mac) ／ iOS ／ Android
関わり
共同開発（2024.01〜）・レベニューシェア／Idle System, Inc. との協業
担当
UI / Graphics、アルバイト採用・マネジメント

01／Technical

技術面

引き続き Unity・C# を用い、UI / Graphics を中心に担当しました。 一方で、本作では「ドメイン（ゲームのロジックや仕様）業務とUI業務を分離して進める」スタイルを採ったことで、 UIの質感やゲーム自体の面白さに納得できない部分を多く残しました。 この反省から、次作 Idle Minertia では MVVM アーキテクチャの採用と、 ドメイン・ゲームデザイン・UIを同じ担当者が貫通する体制へと方針転換しています。

エンジン
Unity
言語
C#
担当領域
UI / Graphics（ドメイン業務は別担当）
反省・学び
UIとドメインを人で分けたことが完成度の停滞を招いた／次作での統合方針につながった
チーム体制
アルバイトを雇用してのリーダー兼プレイヤー的な開発
描画・パフォーマンス
Unity DOTS の低レベル API を用いて 自前の ParticleSystem を実装。 ドメインロジックと連携した大量パーティクルの出力に成功し、 Idle Spiral で抱えていた LineRenderer / CPU起因の描画課題を大きく改善しました。
技術的成果と限界
パフォーマンスは向上したものの、設計面では柔軟性に欠け、継続的な変更を重ねるうえで困難が残りました。 この反省が、次作 Idle Minertia における ECSパターンを参考にした独自設計 と StructLinq ベースの最適化 の採用につながっています。
UI実装
UGUI を使用（プレハブ管理・属人化の課題が残る）

02／Artwork

アートワーク

球体の幾何学的な動きと、ゲーム内通貨と同期する視覚効果を設計しました。 プレステージシステムとランキング画面のUI群が主な担当領域です。ゲーム内スクリーンショットやUI断片は順次追加します。

03／Business Strategy

ビジネス戦略

※本セクションは本人からの追記予定です。 現時点では公開情報とプロジェクト経緯から整理しています。

収益モデル
基本プレイ無料（Free to Play）＋アプリ内購入／DLC
報酬形態
レベニューシェア（収益分配）
配信戦略
Steam (PC) での早期アクセス先行配信、3ヶ月のEA期間を経てモバイルへ展開
長期エンゲージメント
プレステージシステムとランキング機能による長期プレイの動機設計
チーム設計
アルバイトを雇用しての小規模チーム運営（リーダー兼プレイヤー体制）
マーケティング戦略の反省
本作では「ゲームを作ってから、ストア最適化を考える」という順序で進めたため、 スクリーンショット・動画素材の制作や「どう魅せるか」の設計に時間がかかり、 プロモーションの詰めが後手に回る結果となりました。 次作 Idle Minertia ではこの順序を逆転し、広告運用を起点にゲーム体験を逆算する方針へ転換しています。
