_03_／Selected Work

Idle Minertia （旧: Idle Gem）

Idleゲームに Risk of Rain 的なローグライク要素を取り入れた、自動採掘＋装備合成＋ウェーブバトルの放置ゲーム。 企画・開発の大部分を単独で担当し、「遊び手ベースを全てにおいて」という方針のもと、現時点でいちばん自信をもって紹介できる作品です。

リリース
2025年（iOS・Android）／継続アップデート中
プラットフォーム
iOS ／ Android（※旧名「Idle Gem」で公開中の版もあり）
関わり
企画・開発主導（2025.01〜）／Idle System, Inc. と協業
担当
企画・ゲームデザイン・実装・運営・広告運用・コミュニティ運営・リリース作業・アルバイトマネジメント

01／Technical

技術面

Idle Sphere での反省を踏まえ、「ドメインとUIを人で分ける」スタイルから、 「同じ担当者がドメインとゲームデザインとUIを貫通する」体制へと大きく舵を切りました。 アーキテクチャには MVVM を採用し、Model / ViewModel / View の3層構造で Unity への依存性を最小化しています。 レイアウトやスタイルを除くインタラクションは C# 側で定義し、それまでの「ドメイン」を ViewModel 層までに拡張する設計にしました。

エンジン
Unity
言語
C#
アーキテクチャ
MVVM（Model ／ ViewModel ／ View）を採用。 Unity への依存を ViewModel 境界で最小化し、インタラクションロジックを C# 側で一元管理。
開発方針
「遊び手ベースを全てにおいて」— UIの実装者がゲームデザインとドメインまで一貫して担当
体制
メイン開発を2名体制から1名体制に集約し、企画・設計・実装の統合を図った
描画・パフォーマンス
ECSパターン（考え方）を参考にした独自設計を採用。Unity の API に依存せず、 StructLinq を組んで構造体ベースのアロケーションを抑えた処理を構成することで最適化を図っています。 Unity 純正の ECS ほどのスループットは得られないものの、本作の要件には十分な性能を発揮。 Idle Spiral の LineRenderer／Idle Sphere の DOTS 低レベル API／本作の独自ECS と、段階的に最適化の引き出しを増やしてきました。
UI 実装
UIToolkit（UXML） を採用。Idle Spiral・Idle Sphere の UGUI ではプレハブ管理の属人性が課題でしたが、 UXML への移行によって設計の見通し・開発コスト・処理パフォーマンスのすべてで改善が得られました。
継続開発
半年で30回以上のアップデート／25以上の新コンテンツを追加

02／Artwork

アートワーク

採掘演出、装備合成のフィードバック、ウェーブバトルのUI群を設計しています。 アイコン・KV・スクリーンショットは順次追加予定です。本人素材の追加に合わせてこのセクションを拡張します。

03／Business Strategy

ビジネス戦略

これまでと大きく異なる点は、広告運用を起点に、見せ方・打ち出し方から逆算してコンテンツを設計する順序を採ったことです。 前作 Idle Sphere では「ゲームを作ってから、どうストア最適化を図るか」という順序だったため、 スクリーンショット素材や訴求の作り込みに時間がかかり、魅せ方の設計も難しくなりました。 本作ではこの順序を逆転し、どう打ち出すか／どう魅せるかをマーケティング戦略の中心に据え、 UI・ゲームデザイン・プレステージ周回のテンポまでを一貫したコンテンツデザインとして設計しています。

収益モデル
基本プレイ無料（F2P）＋広告表示＋アプリ内購入
価格帯
Remove All Ads ¥800 ／ Diamond Pack ¥150〜¥13,000 ／ Booster +Plus ¥800
設計思想
マーケティング逆算設計： 「どう打ち出すか／どう魅せるか」を先に決め、ゲーム体験のテンポ・ループ・報酬バランスをその訴求に合わせて一貫設計
長期エンゲージメント
プレステージ周回ごとにローグライク的な差分を生み、プレイ体験を毎回異なるものにする
リリース戦略
iOS を先行配信し、Android を後続展開。新機能は継続アップデートで段階投入
ターゲット
放置ゲーム好き × ローグライク／ビルド構築型ゲーム好きの交点
